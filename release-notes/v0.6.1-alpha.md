JsonBadger `0.6.1` consolidates document intake and schema-driven lifecycle behavior across model construction, hydration, migration, and query flows. This release tightens ownership boundaries between `Schema`, `Model`, and `Connection`, simplifies `id_strategy` policy, and updates the docs and tests to match the current runtime.

## What’s Included

1. Unified intake lifecycle: `Model.from(...)` and `Model.hydrate(...)` now build instances, run the composed normalize lifecycle, and rebase tracked state before returning.
2. Schema-owned document shaping: `schema.conform(...)` and `Model#$conform_document(...)` operate on full document envelopes, including top-level base fields.
3. Schema-owned ID strategy: `id_strategy` is selected at schema creation time and consumed downstream through `model.schema.id_strategy`.
4. Capability-aware table migration: `Model.ensure_table()` checks PostgreSQL `uuidv7` support when that strategy is in use.
5. Current JSONB write surface: document and update flows are aligned around the supported update operators such as `$set`, `$unset`, and `$replace_roots`.
6. Synced docs and tests: architecture flow, connection docs, examples, and unit/integration coverage were updated to match the current runtime.

## Release Notes

1. `Connection` no longer participates in `id_strategy` policy. Connection bootstrap is limited to pool creation, capability scanning, and model registry ownership.
2. `Schema` now owns `id_strategy`, `auto_index`, validators, aliases, and the compiled conform tree. `Model` consumes those compiled rules during normalize, validate, and migration paths.
3. Conform, defaults, cast, and validation now run through the model lifecycle instead of being split across older construction seams.
4. Read queries normalize SQL rows into document envelopes before hydration, keeping `find(...)`, `find_one(...)`, `update_one(...)`, and `delete_one(...)` aligned with hydrated model instances.
5. The default schema `id_strategy` now targets `uuidv7`, and the test suite and documentation were updated to reflect UUID-based base-field validation.
