JsonBadger 0.7.1-alpha tightens the document timestamp and envelope contract introduced around the model lifecycle. This patch aligns `Model.create(...)`, `Model.insert_one(...)`, instance saves, and static updates so lifecycle-managed writes are clearly separated from explicit static write gateways.

## Example

```js
const created_user = await User.create({
	name: 'Nell',
	created_at: new Date('2026-03-06T10:00:00.000Z')
});

const inserted_user = await User.insert_one({
	name: 'Draco'
});

const slug = User.schema.get_default_slug();
console.log(created_user.document[slug]);
console.log(inserted_user.document[slug]);
```

> **Note:** Use `Model.create(...)`, `doc.insert()`, or `doc.save()` for lifecycle-managed insert timestamps. Use `Model.insert_one(...)` when you want an explicit static insert that only writes timestamp fields you provide.

## What's Included

1. Direct create write path: `Model.create(...)` now applies lifecycle-managed insert timestamps while calling the insert SQL path directly instead of depending on instance insert as an internal alias.
2. Explicit static insert gateway: `Model.insert_one(...)` no longer auto-generates timestamp fields and only writes `created_at` / `updated_at` when the caller includes them.
3. Timestamp-aware insert SQL: insert query generation now omits timestamp columns unless lifecycle code or caller input provided those base fields.
4. Clear lifecycle contract: `docs/lifecycle.md`, `docs/api/model.md`, and `docs/examples.md` now separate lifecycle-managed writes from explicit static write gateways.
5. Safer envelope examples: documentation now directs direct envelope access through `schema.get_default_slug()` instead of assuming the default slug is always `data`.

## Release Notes

1. `Model.create(...)`, `doc.insert()`, and `doc.save()` on a new document apply lifecycle-managed insert timestamps.
2. Lifecycle inserts preserve caller-provided `created_at`, fill missing `created_at`, and always refresh `updated_at`.
3. `Model.insert_one(...)` is an explicit static write gateway. It writes timestamp fields only when they are present in the input.
4. `Model.update_one(...)` remains an explicit static write gateway. It writes `updated_at` only when the update payload includes it.
5. Instance update paths such as `doc.save()` continue to refresh `updated_at` automatically.
6. Direct document-envelope examples now use `schema.get_default_slug()` so custom default-slug schemas do not copy patterns tied to `document.data`.

## Upgrade Notes

1. Use `Model.create(...)`, `doc.insert()`, or `doc.save()` when you want JsonBadger to manage insert timestamps.
2. Use `Model.insert_one(...)` when you want database defaults or caller-provided timestamp values to control the row.
3. If code reads `document.data` directly, prefer `document[Model.schema.get_default_slug()]` unless the schema explicitly uses `data` as its default slug.
