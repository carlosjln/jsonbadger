JsonBadger `0.5.0-alpha` marks the first complete alpha release of the PostgreSQL JSONB model and query workflow. This release establishes the core schema, model, query, update, and migration surface for working with document-style data on PostgreSQL from Node.js.

## What’s Included

1. Schema-driven models: define schemas with validation, field types, path introspection, index declarations, and document-style runtime behavior.
2. Query and update operators: query JSONB data with operators such as `$gt`, `$in`, `$regex`, `$contains`, `$elem_match`, `$has_key`, `$json_path_exists`, and `$json_path_match`, and update documents with `$set`, `$insert`, and `$set_lax`.
3. Document runtime: work with document instances that support validation, getters and setters, dirty tracking, serialization helpers, `save()`, and instance deletion.
4. Model API: create and query records through `create`, `find`, `find_one`, `find_by_id`, `count_documents`, `update_one`, and `delete_one`.
5. Migration helpers: create tables and indexes with `ensure_table()`, `ensure_index()`, and `ensure_schema()`, with support for first-write table creation.
6. PostgreSQL-specific ID strategies: use `bigserial` by default or native `uuidv7()` when the connected PostgreSQL server supports it.
7. Published docs and examples: the release includes API docs, examples, query translation notes, connection guidance, and local integration testing docs.

## Release Notes

1. This release establishes `id`, `created_at`, and `updated_at` as the standard top-level base fields across the runtime API.
2. Data-returning model operations use document-instance results, keeping reads, writes, serialization, and instance methods aligned.
3. The bundled docs cover the current API surface, query behavior, JSON update operators, connection lifecycle, and local PostgreSQL integration setup.
